# Моделирование динамики социально-эмоциональных взаимодействий в коллективах дискретных автоматов

Данный проект представляет собой программный комплекс для исследования динамики социально-психологической структуры отношений в коллективах агентов. Проект реализован в рамках научно-исследовательской работы факультета Прикладной математики и информатики ТФ МГУ им. М.В. Ломоносова.

Коллектив агентов моделируется как сложная многоагентная система дискретных автоматов, состояние которых изменяется под влиянием эмоциональных факторов, индивидуальных поведенческих архетипов и реляционных связей.

---

## 1. Теоретическое описание модели

Коллектив описывается как кортеж $K = \{A_1, A_2, \dots, A_N\}$, управляемый синхронизирующим автоматом верхнего уровня. Каждый агент $A_i$ представляет собой дискретный автомат со следующим набором параметров:

1. **Вектор эмоционального состояния** $E_i(t) \in [-3, 3]^7 \cap \mathbb{Z}^7$:
   * Печаль — Радость
   * Страх — Спокойствие
   * Гнев — Смирение
   * Отвращение — Принятие
   * Привычка — Удивление
   * Стыд — Уверенность
   * Отчуждение — Открытость (Любовь)

2. **Реляционная матрица направленных связей** для каждой пары $(i, j)$ с целыми значениями на шкале $E_{21} = [-10, 10]$:
   * Выгода ($U_{ij}$) — полезность контакта;
   * Симпатия ($A_{ij}$) — эмоциональная предрасположенность;
   * Доверие ($T_{ij}$) — уровень надежности взаимодействия;
   * Чувствительность к частоте контактов ($R_{ij}$).

3. **Индивидуальный архетип** — неизменяемая во времени стратегия поведения, определяющая функции выбора партнёров, реакцию на отказы, динамику угасания и обновления отношений.
   * *Доступные в GUI архетипы*: Аналитик, Нонконформист, Медиатор, Конкурент, Оппортунист, Протектор, Деструктор, Инноватор, Рефлектор.

4. **Коэффициент чувствительности** $s_i \in (0, 3]$ — параметр, масштабирующий величину и инерционность изменения эмоциональных состояний и реляционных оценок.

### Цикл взаимодействия
На каждом временном шаге (такте симуляции) агенты классифицируют окружение на приоритетных, опциональных и избегаемых партнеров, осуществляют выбор кандидата по правилу Softmax и совершают парный акт общения $\sigma_{ij} \in \{-1, 0, 1\}$ (успех, отказ, неудача), на основании которого пересчитываются эмоциональные векторы и оценки отношений.

## 2. Архитектура и технические особенности

Программный комплекс объединяет высокоуровневое управление на Python и оптимизированный параллельный вычислительный блок на C++:

1. **Параллельное C++ ядро (Engine)**:
   * Реализовано на языке C++ для скоростной обработки $O(N^2)$ взаимодействий;
   * Использует многопоточность **OpenMP** для параллелизации расчётов ежедневного цикла состояний;
   * Связывается с Python-оболочкой через скомпилированные биндинги `emotion_engine`.

2. **Интеграция ClickHouse**:
   * Обеспечивает асинхронное сохранение детальных логов симуляции по дням и слотам (Big Data);
   * Облегчает сбор агрегированной статистики для последующего анализа динамики устойчивости отношений.

3. **Графический интерфейс (GUI)**:
   * Отображает граф взаимосвязей в реальном времени;
   * Содержит детальный Мастер инициализации Университета с автоматическим распределением по факультетам, потокам и группам (1875+ агентов) с расписанием и ограничением вместимости.

## 3. Установка и запуск

### Системные требования
* Python версии 3.10 или новее;
* Компилятор C++ с поддержкой стандарта C++17 и OpenMP (например, `clang`/`gcc`);
* СУБД ClickHouse (опционально, для асинхронного Big Data логирования).

### 1. Автоматическая подготовка окружения
Для создания виртуального окружения и установки необходимых зависимостей:

**Для macOS / Linux:**
```bash
chmod +x install.sh
./install.sh
```

**Для Windows:**
```batch
install.bat
```

### 2. Сборка вычислительного ядра C++
При модификации C++ исходников в папке `core/src/` перекомпиляция осуществляется скриптом:
```bash
chmod +x build.sh
./build.sh
```

### 3. Запуск симулятора
Для старта симулятора с интерактивной графической панелью:
```bash
chmod +x run.sh
./run.sh
```

Вы также можете запустить систему в фоновом (headless) режиме исследований:
```bash
python3 main.py --scenario scenarios_archive/Run03_Harmony_MIXED.json --silent --steps 100
```
